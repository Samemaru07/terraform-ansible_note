HCLは「何を作るか」を記述する**宣言型**の言語。

### 1. ブロック
特定のオブジェクトを定義する器。
```terraform
block_type "label1" "label2" {
	# 内容をここに記述
}
```
- `block_type`: `resource`, `valiable`, `output`, `provider`など。
- `label1`, `label2`: 種類, 名前 (自由) を指定する。リソース定義の際に重要になる。

### 2. 引数
ブロック内の設定値を指定する。
```terraform
key = "value"
```
- 文字列は必ずダブルクォーテーション`""`で囲む。
- 数値やブーリアン, リスト (`[]`), マップ (`{}`) も扱える。

### 3. 識別子
リソースなどを参照する為の名前。
- 文字, 数字, アンダースコア `_`, ダッシュ `-`が使える。
- 戦闘は数字以外である必要がある。

## 記述における重要なルール
- **宣言的である**: 記述した順番通りに実行されるわけではない。Terraformが依存関係 (例: 「ネートワークが出来てからサーバを作る」など) を自動で解析して実行順序を決める。
- **拡張子**: 原則として`.tf`を使う。
