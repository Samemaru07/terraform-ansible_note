- **リソース (Resource)**
	- Terraformにおける最も重要な構成単位。
	- HCLで記述され、プロバイダ (AWSなど) が提供する具体的なサービスを指す。
		- `aws_instance` (サーバ), `aws_vpc` (ネットワーク), `aws_db_instance` (データベース) 。
- **VPC (Virtual Private Cloud)**
	- AWSクラウド内に定義される、ユーザ専用の論理的に隔離された仮想ネットワーク空間。
	- クラウド上の確保する自分専用の「土地」や「区画」
		- サブネット, インターネットゲートウェイ, ルートテーブルなどがこのVPCの中に配置される。
- **CIDR (Classless Inter-Domain Routing)**
	- CIDR (サイダー) は、IPアドレスの割り当てとルーティングを行うための表記法。
	- 表示形式: `x.x.x.x/y`
- **プライベートIPアドレス**
	- インターネットなどの外部ネットワークと直接通信せず、組織内やVPC内などの閉じられたネットワーク内のみで使用されるIPアドレスのこと。グローバルIPアドレスの節約。
	- RFC 1918で指定されている以下の範囲が一般的。

|              範囲               | CIDR表記         | 用途の目安                      |
| :---------------------------: | -------------- | -------------------------- |
|   10.0.0.0 ~ 10.255.255.255   | 10.0.0.0/8     | 大規模なネットワーク (AWS VPCで最も一般的) |
|  172.16.0.0 ~ 172.31.255.255  | 172.16.0.0/12  | 中規模なネットワーク                 |
| 192.168.0.0 ~ 192.168.255.255 | 192.168.0.0/16 | 家庭内や小規模なネットワーク             |
- 予約済みIP
	- また、AWSのVPC内では、予約がる。

|        IPアドレス         | 役割                           |
| :-------------------: | ---------------------------- |
|       10.0.1.0        | ネットワークアドレス                   |
|       10.0.1.1        | VPCルータ (AWSが使用)              |
|       10.0.1.2        | DNSサーバ (AWSが使用)              |
|       10.0.1.3        | 将来の予約用                       |
| 10.0.1.4 ~ 10.0.1.254 | ユーザが利用可能                     |
|      10.0.1.255       | ブロードキャストアドレス (予約はしているが未サポート) |
- **IGW (インターネットゲートウェイ)**
	- VPC という土地の門
		- サーバ (EC2) が「パブリックIP」という名札を持っている場合、IGWはその名札をそのままインターネットへ通す。セキュリティ大事。
		- 1つのVPCに1つしかアタッチできない。
		- NATゲートウェイというのがある。これはVPC→外への一方通行。高額
- **ルートテーブル**
	- サブネットの中にいるサーバが、外に出るための手順が分からない。「外に行きたい時は、あの門 (IGW) を使え」というルールを記した*標識*が必要。

### セキュリティ関連
- **セキュリティグループ** (SG)
	- サーバとの通信を制御する「仮想ファイアウォール機能」
	- **ネットワークACL** がVPC全体のセキュリティを担当する。
- **インバウンドルール** (`ingress`)
	- 外部からサーバへ入ってくる通信の許可設定
	- 単一のポート許可は、`from_port`, `to_port`ともに同じ番号を入れる。
- **アウトバウンドルール** (`egress`)
	- サーバから外部へ出ていく通信の許可設定
	- `from_port = 0`, `to_port = 0`, `protocol = "-1"`で「全てのポート・プロトコル」を意味する。
- **ステートフル**
	- インバウンドで入ってきた通信に対する「戻りの通信」はアウトバンドの設定に関わらず自動的に許可。
		- SSHでログインした際、サーバから自分のPCに結果が返ってくる通信をわざわざ`egress`で定義しなくてよい。通信のセッション状態を一時的に記憶。
